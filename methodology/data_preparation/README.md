# Data Preparation

Scripts used to construct the timestamps and transcripts that define
VSRo-200, starting from raw YouTube podcast videos.

These scripts produce the CSV files published on HuggingFace
(`trainval_annot.csv`, `trainval_auto.csv`, `test_*.csv`). End users
who only want to **use** the dataset should not need to run them —
they should follow [`../../dataset/README.md`](../../dataset/README.md)
instead.

## Pipeline overview

```
        Raw YouTube podcast videos
                   │
                   ▼  Step 1: extract_host_embeddings.py  (one-time per host)
              host_embeddings/*.npy
                   │
                   ▼  Step 2: filter_with_pyannote.py
                   ├─ PySceneDetect        (split into scenes)
                   ├─ InsightFace          (filter: guest only, single face)
                   └─ pyannote             (filter: guest speaks, no overlap)
                   │
                   ▼
            CSV with file_path, start, end (no transcript yet)
                   │
                   ▼  cut clips, then run MultiVSR's preprocess on them
                   │   (see ../../dataset/README.md, Steps 3–4)
                   │
                   ▼
            Face-cropped .avi files (one per clip)
                   │
                   ▼  Step 3: pseudo_label_whisper.py
                              (Whisper-large fine-tuned for Romanian)
                   │
                   ▼
            CSV with file_path, transcript  →  trainval_auto.csv
```

The `trainval_annot.csv` (manual annotations) is produced by humans
post-hoc and is not generated by any script in this folder.

## Files

| File | Purpose |
| --- | --- |
| `extract_host_embeddings.py` | Compute the L2-normalized mean face embedding of a host from a folder of images |
| `filter_with_pyannote.py` | 3-step filter pipeline (scene detection + face-based host removal + audio diarization) |
| `pseudo_label_whisper.py` | Run Whisper-large on each face-cropped clip to produce a pseudo-transcript |

## Step 1 — Extract host embeddings

For each host that may appear in your podcasts (the show's main
presenter), prepare a folder with **5 clear images** of
their face and run:

```bash
python extract_host_embeddings.py \
    --image_dir host_images/host_name/ \
    --output host_embeddings/host_name.npy
```

Repeat per host. The pipeline filter in Step 2 will treat any face
matching one of these embeddings as the host (and reject those scenes,
keeping only guest-speaking moments).

## Step 2 — Filter with pyannote

Produces a CSV of guest-only timestamps for a single video:

```bash
export HF_TOKEN=hf_xxxxxxxxxx        # for pyannote
python filter_with_pyannote.py \
    --video_path videos/abc123.mp4 \
    --host_embeddings host_embeddings/ \
    --output_csv timestamps/abc123.csv
```

## Step 3 — Pseudo-label with Whisper

After cutting the clips and running them through MultiVSR's
preprocessing (see [`../../dataset/README.md`](../../dataset/README.md)),
you'll have face-cropped `.avi` files in a directory tree like
`pycrop/<youtube_id>/<clip_index>/00000.avi`.

Pseudo-transcribe them with the Romanian-tuned Whisper-large:

```bash
python pseudo_label_whisper.py \
    --clips_dir /path/to/face_tracks/all/pycrop/ \
    --output_csv pseudo_labels.csv
```

Output CSV format:

```csv
file_path,transcript
ZWNM2sZxtRg/00001,"așa e în viața noastră"
ZWNM2sZxtRg/00002,"nu pot să cred"
```

This is the format used by `trainval_auto.csv` on HuggingFace (joined
with the timestamps from Step 2 to yield the full
`file_path,start,end,transcript` schema).

## Dependencies

```bash
pip install scenedetect[opencv]
pip install insightface onnxruntime-gpu
pip install pyannote.audio
pip install transformers torch
pip install soundfile pandas tqdm
```

You'll also need `ffmpeg` available on your system PATH.
